# Road vehicles - Software update engineering

**Regulated Area:** Software update / SUMS evidence

**Applicability:** Not a regulation by itself, but supports software update management evidence for OTA and campaign update processes.


## Key Compliance Intent

Provide evidence that vehicle software updates are engineered, deployed and recorded in a controlled manner.

## Primary Vehicle Systems and Components

E/E architecture, ECUs, telematics, gateways, ADAS/automated-driving controllers, diagnostics, software release records, update backend and cybersecurity controls.

## Failure Modes and Symptoms

Corrupted update, wrong software version, lost traceability, failed rollback, unsafe vehicle state during/after update.

## Related Regulations

UN R156; GB 44496; ISO/SAE 21434.

## Engineering Considerations

Added as supporting evidence for SUMS/OTA approval readiness.
